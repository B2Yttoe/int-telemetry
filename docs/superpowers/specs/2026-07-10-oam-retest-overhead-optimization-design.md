# OAM 强制复测开销优化设计

## 目标

在不修改第一阶段卫星网络真值模型的前提下，降低增强版 LEO-INT-MC 的 OAM 强制复测开销，并完整重跑实验2。三种星座的增强版遥测开销均不得高于各自增强前 INT-MC；相对当前增强版，CPU、队列、电量和链路利用率 MAE 的退化不得超过 1%，节点模式和链路状态准确率下降不得超过 0.1 个百分点。

验收条件为：

\[
B_{enhanced,new}(s) \leq B_{before}(s),\qquad s\in\{Iridium,Telesat,Starlink\}
\]

\[
MAE_{new}(s,m) \leq 1.01\,MAE_{enhanced,current}(s,m)
\]

其中 \(m\) 包含 CPU、队列、电量和链路利用率。准确率指标满足：

\[
Accuracy_{new} \geq Accuracy_{enhanced,current}-0.001
\]

## 当前问题

Starlink 1584 的当前增强版在 48 个时间片中选择 513 条 probe 路径，其中 126 条命中 OAM 强制复测目标。强制路径只占 24.56%，却产生 693,189.12 B，占总遥测生成字节的 37.14%。其平均路径包含 6.60 个节点，普通路径平均包含 3.60 个节点；强制路径平均产生 5,501.50 B，普通路径平均产生 3,031.64 B。

增强版的总字节为 1,866,433.344 B，比增强前多 46,081.344 B。只需消除当前强制复测开销的约 6.65%，即可使 Starlink 回到增强前的 23.9419 B/节点/时间片以内。

当前选择器存在两个主要冗余来源：

1. 候选路径只要命中强制目标就获得高优先级，即使该目标已被先前选中路径覆盖，后续路径仍可能继续获得强制加分和预算突破资格。
2. 强制路径通常采用完整 metadata 和 all-adjacent 观测；沿途非目标节点也会扫描相邻链路，生成与复测目标无直接关系的记录。

## 设计原则

1. 先删除不改变关键目标直接观测的冗余，再考虑减少复测目标数量。
2. 目标节点和目标链路保持完整 metadata，优先压缩纯转发节点和非目标相邻链路。
3. OAM 强制覆盖只对尚未覆盖的目标生效，已覆盖目标不再获得强制预算突破。
4. 每轮只引入一类节省机制，并使用相同真值、业务、候选路径和反馈输入重跑，保证因果关系可解释。
5. 任何误差或准确率越过验收门限时，撤销该轮新增的节省强度，而不是放宽质量门限。
6. 当前反馈输入在本轮保持不变，用于隔离验证 OAM 执行机制。真值辅助反馈与可部署 OAM-only 反馈将在后续报告中拆分，不能把 oracle 反馈描述为真实部署能力。

## 方案一：强制目标边际去重

选择器在每个时间片维护已覆盖的 OAM 强制目标集合 \(C_t^{OAM}\)。候选路径 \(p\) 的新增强制目标为：

\[
\Delta R(p)=R(p)\setminus C_t^{OAM}
\]

只有 \(|\Delta R(p)|>0\) 时，路径才获得强制选择加分和字节预算突破资格。路径被选中后执行：

\[
C_t^{OAM}\leftarrow C_t^{OAM}\cup R(p)
\]

候选路径的 OAM 边际价值按单位遥测字节计算：

\[
Score_{OAM}(p)=\frac{\sum_{i\in\Delta R(p)}q_i}{B(p)}
\]

其中 \(q_i\) 是目标优先级，\(B(p)\) 是路径预计生成字节。该机制不删除任何复测目标，只减少同一目标的重复路径覆盖。

## 方案二：目标感知 metadata 与观测范围

每条强制路径输出节点目标和链路目标标识。probe runner 按记录是否命中复测目标选择 metadata 配置：

\[
B_h=\begin{cases}
96\text{ B}, & h\text{ 命中强制节点或链路目标}\
88\text{ B}, & h\text{ 仅承担转发}
\end{cases}
\]

目标节点保留完整节点状态；目标链路保留完整链路状态。纯转发节点保留节点编号、时间、入端口、出端口和转发所需基础状态，不扫描全部非目标相邻链路。若某个转发节点同时是目标链路端点，则允许扫描该目标链路，但不扩展到其余无关端口。

report 大小继续按实际逐条记录的 `hop_metadata_bytes` 求和，选择器的预计成本和 runner 的实际成本必须使用同一配置，避免预算估算与运行结果不一致。

## 方案三：目标聚合与软复测

只有前两项仍无法满足字节目标时才启用本方案。

候选路径优先覆盖多个尚未覆盖的邻近目标：

\[
Score(p)=\frac{\sum_{i\in\Delta R(p)}q_i+I_{node}(p)+I_{link}(p)}{B(p)}
\]

普通低置信度和轻度 stale 目标降为软复测；unknown、高冲突、关键拓扑变化和超过 AoI 截止时间的目标保持硬强制。软复测允许在未来两个时间片内完成，并设置两个时间片冷却期；状态突变、冲突升级或 up/down 转换可突破冷却期。

本方案不得在线使用第一阶段真值误差决定强制等级。第一阶段真值只用于离线阈值校准和实验后评价。

## 数据流

1. 第一遍 INT-MC 生成 Ground OAM 重构、控制动作和复测反馈。
2. 增强选择器读取与当前正式实验相同的反馈输入。
3. 拓扑复用缓存提供历史 probe plan，选择器在当前活动图上验证和局部修复。
4. 选择器按尚未覆盖的 OAM 目标计算边际增益，输出目标感知 probe 字段。
5. probe runner 对目标记录使用完整 metadata，对纯转发记录使用轻量 metadata，并只扫描目标相关相邻链路。
6. Ground OAM 接收 report，完成直接重构和矩阵补全。
7. 评价器与当前增强版、增强前 INT-MC 比较开销、误差、准确率和覆盖质量。

## 实验迭代顺序

### 第一轮

只启用强制目标边际去重和目标感知 metadata。强制目标集合、反馈输入、采样率和最大路径数保持不变。

### 第二轮

若任一星座仍高于增强前字节，启用多目标路径聚合，同时保持硬目标数量不变。

### 第三轮

仅在前两轮仍未达标时启用软复测、两片冷却和限期调度。每次只调整一个参数，并在 Starlink 48片运行后检查 1% 非劣化约束。

达到 Starlink 目标后，必须完整重跑 Iridium 66、Telesat 351 和 Starlink 1584，不能只用大型结果推断中小规模表现。

## 测试

行为测试必须覆盖：

1. 同一强制目标被第一条路径覆盖后，第二条只命中相同目标的路径不再获得强制加分。
2. 一条路径覆盖两个尚未覆盖目标时，其边际价值高于只覆盖一个目标的等成本路径。
3. 目标节点和目标链路记录保持 96 B；纯转发记录使用 88 B。
4. 纯转发节点不生成无关 `local-adjacent-link` 记录。
5. report 字节等于 report header 加所有实际记录 metadata 字节。
6. 非 OAM 普通路径行为与当前实现一致。
7. 实验结果校验器同时检查字节目标、四项 MAE 1%门限和两项准确率门限。

## 输出与归档

当前 `reports/experiment2-int-mc-energy-physics-final` 保留不动。新结果写入新的时间戳或明确版本目录，完成验收后再更新正式实验2综合报告。报告必须分别列出：

- 普通路径与 OAM 强制路径数量和字节；
- 目标记录、转发记录和相邻扫描记录；
- 重复目标覆盖抑制数量；
- metadata 节省字节；
- 三种规模的增强前、当前增强版和新增强版；
- 每项误差相对当前增强版的变化率；
- 未达标项及对应回退机制。
