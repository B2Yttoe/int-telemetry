# 实验 12：拓扑版本化严格因果等预算实验设计

## 1. 研究目标

实验 12 只验证面向中大型动态 LEO 星座的核心创新：在相同候选路径、相同 INT 观测字节硬预算、相同 Ground OAM 输入和相同状态重构器下，拓扑类复用、拓扑版本目标、局部修复与风险预测是否分别带来可测量的边际收益。

本实验不要求增强方案在所有星座、所有状态指标和所有开销维度上全面胜出。正式结论限定为：在 Telesat-1015 中型星座和 Starlink 主壳层大型星座中，增强机制能否改善规划计算开销、无效路径比例、单位开销有效覆盖或关键重构误差中的一项或多项，同时不突破相同遥测字节预算。

## 2. 公平性边界

所有实验档位固定以下变量：

- 第一阶段真值与动态性扰动；
- 候选 probe 路径集合；
- INT metadata 字段和单跳字节；
- 每节点每时间片遥测字节硬预算；
- Ground OAM 输入及一时间片反馈滞后；
- 矩阵补全、节点状态补全和物理后处理配置；
- 时间片数量、随机种子与报告回传预算。

第一阶段真值只能被离线评价器用于计算 MAE、准确率和路径合法性，不得进入路径选择、OAM 反馈、风险评分或补全输入。

## 3. 累计机制阶梯

实验采用相邻档位只增加一个机制的累计设计：

1. `fresh-replan`：每片重新规划，不复用拓扑类，不启用版本目标、局部修复和风险预测。
2. `topology-class-reuse`：只增加拓扑类识别和 probe plan 复用。
3. `topology-versioned-objective`：在拓扑类复用上增加 `class_id@signature` 版本标识和单位开销信息收益目标。
4. `topology-local-repair`：再增加对失效路径局部替换和变化链路增量探测。
5. `topology-risk-prediction`：再增加基于可预测轨道、距离、极区和接触窗口的未来断链风险评分。

相邻两档的差值作为对应机制在当前依赖链上的边际贡献。它不是完全因子化的独立因果效应；正式论文如需更强证据，可在同一运行器中追加围绕完整方案的 leave-one-out 模式。

## 4. 严格时间因果在线回放

时间片 `t` 的路径规划只允许使用：

- 不晚于 `t-1` 的 OAM 重构、置信度、冲突和控制动作；
- 由 TLE/SGP4 或 Walker 星历确定性推算的当前及未来轨道与接触关系；
- 固定的系统配置、天线约束和遥测预算。

禁止使用：

- 时间片 `t` 或未来时间片的节点、链路真值状态；
- 由 `|estimate - truth|` 生成的 oracle 反馈；
- 未来 OAM 报告或在当前片尚未完成回传的报告；
- 离线全时间序列统计得到的隐藏状态热点。

因果审计必须逐行验证：

\[
t_{source} < t_{target}
\]

并验证 `feedback_lag_slices >= 1`、`truth_error_feedback_enabled = false`、选择器报告中的因果违规数为零。额外执行未来反馈反事实：向截止片之后注入反馈，截止片之前的 probe plan 哈希必须保持不变。

## 5. 等预算指标

主约束为：

\[
C_{bytes}(P_t) \le B_t
\]

并使用字节填充到硬上限，保证各方案实际遥测字节一致。默认预算沿用实验 10 的冻结校准值：

- Telesat-1015：`435.7265 B/node/slice`；
- Starlink 主壳层：`21.3359 B/node/slice`。

主要结果分为四类：

- 规划代价：规划墙钟时间、fresh replan 数、复用命中率、局部修复数；
- 网络代价：遥测字节、遥测能量、报告回传字节；
- 测量质量：直接节点/链路覆盖、活动链路覆盖、无效 probe 路径比例；
- 重构质量：CPU、队列、电量和链路利用率 MAE，节点模式与链路状态准确率。

## 6. 运行范围

低成本冒烟运行使用两个星座、25% 动态性压力和少量时间片，目的是验证机制隔离、公平预算和因果审计能正确工作，不据此形成论文统计结论。

正式运行使用两个星座、48 个时间片、至少三个动态性档位和多随机种子，并报告均值、95% 置信区间与配对效应量。正式运行不再包含 Iridium 小型星座，避免把不符合论文主张范围的结果混入核心结论。

## 7. 产物

- `experiment12-summary.csv`：星座、机制档位与核心指标；
- `experiment12-mechanism-deltas.csv`：相邻档位的边际变化；
- `experiment12-causal-audit.csv`：逐运行因果审计；
- `experiment12-results.json`：完整机器可读结果；
- `EXPERIMENT_12_REPORT.md` 和 `index.html`：中文实验说明、图表与结论边界。

## 8. 验收条件

1. 两个星座的五档机制全部完成，且每档使用同一硬字节预算。
2. 相邻档位除目标机制外，其余机制和重构配置一致。
3. 所有因果审计均通过，违规数为零，且不存在 truth-error feedback。
4. 报告明确区分冒烟结果与正式统计结论。
5. 报告不宣称增强方案在所有指标全面优于基线，只陈述有数据支持的中大型等预算优势。
