# Scale-Adaptive Budget-Neutral LEO INT-MC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the fixed absolute probe-path cap with a causal, coverage-derived shared byte budget for native and enhanced INT-MC, then make enhanced importance repair budget-neutral with strict forward-only transit and orbit-plane representative rotation.

**Architecture:** A new pure budget module estimates per-slice node/link coverage demand, a deterministic greedy coverage witness, a standard-metadata byte cap and a safety path cap. The selector consumes that shared result in both native and enhanced modes; enhanced mode replaces low-value base paths inside the cap and emits explicit target masks, while the runner enforces the selector's per-slice cap using actual bytes.

**Tech Stack:** Node.js ES modules, CSV/JSON experiment artifacts, executable `node:assert` tests, existing INT-MC selector/runner/OAM pipeline.

## Global Constraints

- Do not modify Stage 1 truth generation.
- Runtime planning must not consume hidden current-slice Stage 1 state.
- Native and enhanced variants must receive identical per-slice `scale_budget_bytes` and `scale_budget_safe_path_cap` for the same input.
- No constellation-name-specific branches.
- Enhanced replacement must not increase selected path count or exceed the shared byte budget.
- Non-target transit hops must forward but write zero metadata in strict mode.
- Infeasible AoI must be reported, not hidden by unbounded probes.
