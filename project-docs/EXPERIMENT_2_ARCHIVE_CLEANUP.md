The Experiment 2 archive had accumulated many intermediate, smoke, failed, and repeated rerun outputs. Most of the disk usage came from full per-constellation run artifacts, while the representative evidence needed for later analysis is stored in top-level CSV/JSON/HTML/Markdown reports.

## Second Cleanup: Slim Representative Archive

The 2026-07-11 cleanup retained the representative Experiment 2 versions but reduced the latest formal run to an evidence-oriented archive. The following three milestones remain available as compact top-level reports:

- `energy-physics-final`: representative energy-model milestone.
- `oam-optimized-final`: representative OAM overhead-control milestone.
- `oracle-free-replay`: final deployable, no-truth-leakage milestone and the recommended Experiment 2 result.

For `reports/experiment2-int-mc-oracle-free-replay`, the cleanup preserved:

- comparison reports in CSV, JSON, HTML, and Markdown;
- per-constellation INT-MC evaluation JSON;
- node and link error CSV files;
- probe paths, probe summaries, coverage, and per-slice overhead;
- contact-plan summaries, windows, and forecast evaluation;
- OAM control actions and deployable retest feedback;
- all small manifests and run reports needed to identify the experiment configuration.

The cleanup removed 54 reproducible bulk artifacts:

- full Ground OAM estimate graphs;
- full probe sampling masks;
- full reconstructed node and link matrices;
- duplicated Ground OAM reconstructed node and link tables;
- full contact-plan JSON/CSV details while retaining their summaries and windows;
- per-hop INT records while retaining probe paths, probe summaries, and overhead summaries.

These files are generated intermediate products rather than unique experimental conclusions. They can be regenerated by rerunning the Experiment 2 scripts from the retained configuration and source code.
