# 实验 14B：真实性声明与内部潜变量边界

## 1. 声明原则

实验 14B 将项目数据分成两类：

1. **公开可观测量**：GP/OMM 轨道、用户侧 RTT、NDT7 用户吞吐、Cloudflare Radar 业务时序。此类变量可以与外部数据进行独立检验。
2. **运营商内部潜变量**：逐星 CPU、电量、队列、真实 ISL 调度和逐流状态。此类变量没有公开逐星真值，只能检验物理约束、守恒关系、参数来源和相对实验效果。

因此，项目允许声明：

> CPU、电量和队列是业务、转发、遥测、链路负载和光照共同驱动的物理公式仿真状态，可作为同一仿真环境下比较遥测算法的真值。

项目不允许声明：

> 这些状态与 Starlink 运营商某颗卫星的真实内部寄存器或设备遥测逐点一致。

## 2. CPU 状态

运行模式下的 CPU 利用率由不同工作来源相加：

\[
U_i^{cpu}(t)=
U_i^{source}(t)+
U_i^{transit}(t)+
U_i^{queue}(t)+
U_i^{telemetry}(t)
\]

其中源任务、经过流量、队列处理和遥测处理均由输入业务或网络事件触发。空业务时，业务相关 CPU 项应为零；控制面基础功耗不等同于业务 CPU 利用率。

CPU 状态的可信度来自：

- 输入任务和转发字节与 CPU 增量之间存在确定映射；
- 利用率被限制在设备容量范围；
- 相同输入、配置和随机种子得到相同结果；
- INT/INT-MC 只能观测或恢复该状态，不能在正式算法中读取隐藏真值。

## 3. 队列状态

链路或节点队列必须满足离散流量守恒：

\[
Q_e(t+1)=
\min\left[
Q_e^{max},
\max\left(0,Q_e(t)+A_e(t)-S_e(t)\right)
\right]
\]

其中：

- \(A_e(t)\) 是时间片内到达流量；
- \(S_e(t)\) 是链路有效容量实际服务的流量；
- 超过 \(Q_e^{max}\) 的部分单独计为丢弃；
- topology-down 链路服务量为零。

实验 13 已使用 ns-3 对相同输入下的逐包排队、MTU、丢包和 report 交付进行交叉验证。该证据用于检查聚合模型趋势和适用边界，不把聚合队列绝对值包装成真实硬件测量。

## 4. 电量状态

每颗卫星维护电池能量：

\[
E_i(t+\Delta t)=
\operatorname{clip}\left[
E_i(t)+
\eta_{ch}\max(P_i^{gen}-P_i^{load},0)\Delta t-
\frac{\max(P_i^{load}-P_i^{gen},0)}{\eta_{dis}}\Delta t
\right]
\]

负载功率由基础平台、通信、计算、载荷和遥测组成：

\[
P_i^{load}=P_i^{base}+P_i^{comm}+P_i^{compute}+P_i^{payload}+P_i^{telemetry}
\]

光照区发电、阴影区放电，电池状态必须满足：

\[
E_i^{min}\le E_i(t)\le E_i^{max}
\]

电量可信度应通过能量平衡残差、充放电斜率、阴影持续时间和参数敏感性报告，而不是通过不存在的运营商逐星电池数据进行虚假外部拟合。

## 5. ISL 拓扑边界

项目中的星间链路是公开轨道数据与下列约束生成的**潜在/调度 ISL 网络**：

- 轨内相邻关系；
- 轨间候选关系；
- 最大通信距离；
- 地球遮挡；
- 极区限制；
- 天线端口和每星最大链路数；
- 太阳规避和链路预算。

公开 GP/OMM 能验证卫星位置，不能给出 Starlink 运营商每个时间片真实启用的激光链路。因此论文和数据集应使用 `candidate/scheduled ISL`，不能写成 `operator-observed ISL`。

## 6. 外部验证与因果要求

正式外部验证必须满足：

- 代码、配置和参数先冻结并保存 SHA-256；
- 测试时间片与训练时间片不重叠；
- 测试目标值不参与参数拟合或区间校准；
- 测试后参数更新次数为零；
- 成功和偏差同时报告；
- 缺少 token 或未来窗口尚未结束时标记 `pending`，不得使用旧数据或合成数据替代。

## 7. 可用于论文的结论

完成实验 14B 后，可支持以下结论：

> 模型在新鲜公开轨道输入、同口径用户性能输出和冻结后的外部测试条件下，对公开可观测量具有可量化的一致性；内部节点状态由明确物理与业务方程产生，适合作为同一环境下比较 INT/OAM 重构算法的仿真真值。

仍不能支持以下结论：

> 模型复刻了 Starlink 运营商真实 ISL 调度、星上 CPU、电池、队列和用户逐流运营数据。
